# 领域包 · 培训课程 / 教材

## 推荐 Level

L2-L3。课程体系化后通常 L3。

## 启动问用户

1. 培训对象？（新员工 / 在职提升 / 外部客户）
2. 课程形式？（面授 / 录播 / 混合 / 书面）
3. 多少门课？几个层级？

## 定制化目录

```
10-curriculum/
  ├── 00-课程体系.md            整体框架 / 学习路径图
  ├── 10-<课程1>/
  │   ├── 00-大纲.md
  │   ├── 10-<模块1>.md
  │   └── 11-<模块2>.md
  ├── 11-<课程2>/ ...
  ├── 20-modules/                跨课程的通用模块
  └── 90-assessments/             测评题库

20-specs/
  ├── 00-教学标准.md            课时 / 难度 / 评估方式
  └── 10-讲师手册.md

30-resources/
  ├── 10-lessons.csv             课程索引（SSoT）
  ├── 11-lessons-changelog.md
  ├── slides/                     课件
  └── exercises/                  练习题
```

## 常见输入分拣

| 输入 | 处置 |
|---|---|
| 学员反馈 | inbox/feedback/ → 修订大纲 |
| 行业新知识 | inbox/new-knowledge/ → 可能加入某课程 |
| 测评错题分析 | inbox/assessments/ → 修订题目或讲解 |
| 课程结构调整 | 建 ADR |

## 典型 ADR

- ADR-0001：课程体系设计
- ADR-0002：某课程大纲重构
- ADR-0003：评估标准调整

## CLAUDE.md 片段

```markdown
## 项目专属约定

- `30-resources/10-lessons.csv` 是课程索引真源
- 学员反馈周度汇总,决定是否修订
- 大纲调整必须 ADR
```
