# 领域包 · 学术论文 / 研究

> 任何学术论文写作项目（会议 / 期刊投稿 / 毕业论文 / 技术报告）的 quick-start。

## 推荐 Level

**通常是 L2-L3**：
- L2 如果是短期会议投稿（3 个月内）
- L3 如果是长期项目（毕业论文 / 多轮投稿修改）

很少到 L4（论文通常没有"回退不起"的下游）。

## 启动时问用户的 3 个问题

1. 目标发表渠道？（决定字数 / 模板 / 规范：CHI / NeurIPS / ACL / 期刊 / 毕业论文）
2. 几个作者？（独立 / 合作）
3. 现在处于哪个阶段？（找题 / 做实验 / 写作中 / 投稿后修改）

## 定制化目录

```
10-chapters/                 论文主体（按章节）
  ├── 00-大纲.md              论文整体结构与论点
  ├── 01-intro.md
  ├── 02-related-work.md
  ├── 03-method.md
  ├── 04-experiments.md
  ├── 05-discussion.md
  └── 06-conclusion.md

10-arguments/                核心论点（可选,按论点平行组织）
  ├── 00-论点树.md
  └── <argument-1>.md ...

20-specs/                    写作规范
  ├── 00-投稿要求.md           CHI 字数 / 格式 / 截稿日期
  ├── 10-引用规范.md            BibTeX 样式
  └── 20-图表规范.md

30-resources/                资源
  ├── references.bib           参考文献（SSoT）
  ├── 11-references-changelog.md
  ├── figures/                  图
  ├── tables/                   表
  └── experiments/              实验数据

40-process/
  ├── inbox/                   新读论文 / 审稿意见 / 讨论
  └── decisions/
      └── ADR-0001-研究问题定义.md
      └── ADR-XXXX-方法论选择.md
      └── ADR-XXXX-实验设计修订.md

99-raw/                      原始材料
  └── 读过的论文 PDF / 原始数据
```

## 常见输入分拣

| 输入类型 | 处置 |
|---|---|
| 读到的相关论文 PDF | `99-raw/papers/` + 加进 `references.bib` |
| 实验原始数据 | `99-raw/data/` |
| 审稿意见（投稿后返回）| `40-process/inbox/YYYY-MM-DD-review-<venue>.md` |
| 和导师 / 合作者的讨论 | `40-process/inbox/YYYY-MM-DD-meeting.md` |
| 新想到的论点 | `10-arguments/` 新增文件 或 inbox 先记 |
| 方法或实验的调整 | 建 ADR |

## 典型 ADR 主题

- ADR-0001：研究问题的定义
- ADR-0002：方法论选择（为什么用 X 不用 Y）
- ADR-0003：实验设计（被试数 / 统计方法）
- ADR-0004：审稿意见回应策略
- ADR-0005：重大修订（换投期刊 / 换 framing）

## 特有机制

### 论点一致性检查

每次改方法或实验结论,AI 主动扫所有章节的相关表述,报告不一致处。

### BibTeX 管理

- 新读论文 → 立即进 `references.bib`
- 未引用的 reference 每月清理
- 避免同一篇论文多次登记

### 审稿意见处理

投稿后返回的审稿意见作为特殊 inbox 条目,AI 协助：
1. 分类：major / minor / 误解 / 新实验
2. 生成 response letter 模板
3. 根据每条意见生成对应的修改 checklist

## CLAUDE.md 片段

```markdown
## 项目专属约定

- 目标投稿：<CHI 2026 / 具体期刊>
- 字数上限：<12 页 / 8000 词>
- 截稿日期：<YYYY-MM-DD>
- 引用样式：<ACM / IEEE / APA>
- `30-resources/references.bib` 是文献真源,不在 .md 里重复信息

## 输入分拣

| 输入类型 | 处置 |
|---|---|
| 新读到的论文 | 99-raw/papers/ + references.bib |
| 实验数据 | 99-raw/data/ |
| 审稿意见 | inbox/YYYY-MM-DD-review-<venue>.md |
| 讨论 / 灵感 | inbox/YYYY-MM-DD-主题.md |
| 方法 / 实验调整 | 建 ADR |

## 论点一致性

改 method / experiment 的结论时,AI 扫全文找不一致表述。
```

## 时间节点提醒

论文项目的强时间约束,AI 主动提醒：
- 截稿前 4 周：方法 + 实验必须定稿
- 截稿前 2 周：初稿完成
- 截稿前 1 周：润色 + figure 打磨
- 截稿前 3 天：最终检查清单
