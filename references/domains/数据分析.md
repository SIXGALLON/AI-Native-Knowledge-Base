# 领域包 · 数据分析报告

## 推荐 Level

L2-L3。单次报告 L2,持续的 BI 报告体系 L3。

## 启动问用户

1. 一次性专题报告还是持续性 BI 报告？
2. 数据来源是什么？（产品数仓 / 实验平台 / 第三方）
3. 报告给谁用？（高管决策 / 产品迭代 / 技术审视）

## 定制化目录

```
10-findings/
  ├── 00-总览.md                本次/本期的核心结论
  ├── 10-<主题1>.md              按主题拆分析
  ├── 11-<主题2>.md
  └── 90-cross-analysis.md       跨主题洞察

20-specs/
  ├── 00-分析方法论.md           指标定义 / 统计方法 / 置信区间
  └── 10-可视化规范.md

30-resources/
  ├── 10-data-index.csv           数据集索引（SSoT）
  ├── 11-data-changelog.md
  ├── datasets/                    清洗后的数据
  ├── queries/                     SQL / 分析脚本
  ├── charts/                      图表（可复用的图像）
  └── notebooks/                   Jupyter / Colab

99-raw/
  ├── raw-exports/                 原始数据导出
  └── source-definitions/          数据源定义
```

## 常见输入分拣

| 输入 | 处置 |
|---|---|
| 原始数据导出 | `99-raw/raw-exports/` |
| 新的分析需求 | inbox/ → 判断是否触发新 study |
| 数据口径问题 | inbox/ → 可能更新方法论文档 |
| 业务方提问 | inbox/questions/ |
| 方法论调整 | 建 ADR |

## 典型 ADR

- ADR-0001：核心指标定义
- ADR-0002：数据源切换
- ADR-0003：统计方法变更
- ADR-0004：口径修订（如 DAU 定义变化）

## 特有机制

### 指标真源

所有指标定义必须在 `20-specs/00-分析方法论.md` 里唯一定义,任何文档引用直接链接到该处。

### 数据版本

每个分析结论引用的数据必须标明：
- 数据源（表名 / API）
- 时间范围
- 提取日期
- 口径版本
