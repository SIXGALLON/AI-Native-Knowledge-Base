# Inbox · 项目唯一输入入口（模板）

> 所有散乱输入（会议纪要 / 群消息 / 临时要求 / 标注员疑问 / 灵感）都扔这里。
> AI 协助分拣到正式位置。**不允许**原始输入散落在项目根目录。

---

## 文件命名约定

```
YYYY-MM-DD-<主题关键词>.md         一般输入
YYYY-MM-DD-meeting-<主题>.md       会议纪要
YYYY-MM-DD-wechat-<主题>.md        群聊提取
YYYY-MM-DD-issue-<N>.md            问题咨询
```

日期一定要有，主题简短。

---

## 分拣流程（AI 拿到输入后）

```
用户扔来一段内容
  ↓
AI 判断内容类型：
  ├─ 完全原始（录音稿 / 他人原文 / docx 节选）
  │   → 存进 99-raw/ （只读保护）
  │
  ├─ 会议纪要 / 群消息 / 用户反馈
  │   → 按上方命名约定存进 inbox/
  │   → 提炼"可行动项"列表给 Owner 确认
  │
  ├─ 明确的规则变更 / 定义修订
  │   → 起草 ADR + 改 SSoT + 联动检查
  │   → 原始对话存进 inbox/ 留痕
  │
  └─ 不明确的请求
      → 问用户：这是原始记录还是想落地的结论？
```

## 会议纪要模板

```markdown
# YYYY-MM-DD · <会议主题>

## 参会

- <name 1>
- <name 2>

## 背景

<为什么开这个会>

## 讨论要点

### 议题 1：<>

- 共识：
- 分歧：
- 决策：

### 议题 2：<>

...

## 可行动项（Action Items）

- [ ] <动作 1> @ <owner> · DDL <date>
- [ ] <动作 2> @ <owner> · DDL <date>

## 关联文件 / 后续 ADR

- <link>
- ADR-XXXX（如产生方向性决策）
```

---

## 定期清理

- **每周末**：owner 和 AI 一起过一遍 inbox，把已落地的移到 `inbox/archive/`
- **每月**：如果 inbox 里某份文件超过 30 天仍没动作，触发讨论是"不重要可删"还是"漏做了"

---

## 不要把以下内容放 inbox

- 正式 ADR（放 `40-process/decisions/`）
- 正式规则 / 定义（放 `20-rules/` 或 `00-charter/`）
- 原始材料（放 `99-raw/`，不可编辑）
- 派生物（Excel / 图 / 报告 —— 放它们的归属目录）

---

## inbox 是**临时暂存区**，不是永久文件夹

核心原则：**inbox 里的文件应该有倾向于 "被分拣到别处" 的命运**。如果某份文件永远躺 inbox，说明它需要被搬到正式位置或被删除。
